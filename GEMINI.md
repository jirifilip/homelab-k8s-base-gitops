# GEMINI.md - Project Context

## Project Overview
This repository is a **FluxCD (v2) GitOps** project for managing a Kubernetes homelab cluster. It contains the base configuration for Flux components and the synchronization settings to track changes in this repository.

### Main Technologies
- **FluxCD (v2):** GitOps toolkit for Kubernetes.
- **Kubernetes:** Container orchestration platform.
- **Kustomize:** Tool for template-free customization of YAML manifests.
- **GitOps:** Operational model where the Git repository is the single source of truth for infrastructure.

### Repository Structure
- `flux-system/`: Contains the core Flux components (`gotk-components.yaml`), synchronization resources (`gotk-sync.yaml`), and a Kustomization base (`kustomization.yaml`).
- `.idea/`: Project-specific settings for JetBrains IDEs.

## Operations and Management

### Syncing and Reconciliation
Flux automatically reconciles the cluster state with this repository every 10 minutes (as per `gotk-sync.yaml`). You can trigger a manual reconciliation using the `flux` CLI:

```bash
# Reconcile the flux-system kustomization
flux reconcile kustomization flux-system
```

### Checking Status
Monitor the status of Flux components and synchronizations:

```bash
# Check all Flux resources
flux get all

# Check source synchronization
flux get sources git

# Check kustomization status
flux get kustomizations
```

### Installation / Bootstrap
The project was likely bootstrapped using the `flux bootstrap` command. If you need to reinstall or update the core components:

```bash
# Example bootstrap command (adjust as needed)
flux bootstrap github \
  --owner=jirifilip \
  --repository=homelab-k8s-base-gitops \
  --branch=main \
  --path=./flux-system
```

## Development Conventions

- **GitOps Workflow:** Any changes made to the `main` branch will be automatically applied to the Kubernetes cluster by Flux.
- **Manual Edits:** Do not manually edit files marked with `# This manifest was generated by flux. DO NOT EDIT.` (e.g., `gotk-sync.yaml`) unless you are performing a manual migration or reconfiguration.
- **Adding New Resources:** When adding new applications or infrastructure components, it is recommended to create a dedicated directory (e.g., `apps/` or `infrastructure/`) and reference it in the `flux-system` kustomization or create new `Kustomization` resources.
- **Kustomize Usage:** This project uses Kustomize for resource management. Prefer using Kustomize overlays for environment-specific configurations.
